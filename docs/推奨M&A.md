推奨M&A v0.0.1
Recommended Buy-Side Filter for Owner SME Acquisition
	0.	Scope

本仕様は、後継者不足のオーナー系中小企業を対象とした「勝ち寄りM&A案件」の判定ルールを定義する。
財務・顧客構造・許認可・価格・レバレッジ条件を定量化し、買収可否と優先度ラベルを決める。
	1.	記号

S  : 年間売上高
GM : 粗利率 (gross margin, %)
E  : EBITDA（ノーマライズ後）
NI : 税引後利益（ノーマライズ後）
C  : 内部留保のうち自由に使える実質キャッシュ相当額
D  : 有利子負債残高（既存分）
P  : 株価（エクイティバリュー、買収対価）
r  : 借入金利（実効金利）
D_new   : 新規に調達する有利子負債
D_total : 買収後の有利子負債合計 = D + D_new
DSCR    : Debt Service Coverage Ratio の近似 E / 利払費用
DSCR_target_hard : 安全側に見る目標DSCR (推奨値 3.0)
	2.	対象企業の前提イメージ

・オーナー系中小企業
・首都圏（地域制約がある場合）
・後継者不足による売却意向
・事業としては継続可能で、かつ許認可ビジネスを想定
	3.	ハード条件 (必須フィルタ)

以下を全て満たさない場合、価格検討前に原則見送りとする。

H1. 黒字継続
過去10年以上、経常利益ベースで黒字継続していること。

H2. 粗利率
GM >= 40%

H3. 長期B2B比率
売上のうち、長期B2B契約（1年以上継続見込み）の比率 >= 60%

H4. 有利子負債
D == 0  または  D <= E × 1.0
（既存有利子負債はEBITDAの1年分未満）

H5. 許認可・免許
許認可・免許が「オーナー個人資格」ではなく「法人または継続可能な体制」に紐づいていること。
オーナー退任後も、要件（有資格者数等）を満たせる見込みがあること。

H6. 顧客集中
任意の単一取引先の売上比率 < 40%
（上位1社が40%を超える場合は原則NG。例外扱いする場合はディール個別評価とする。）
	4.	価格条件 (バリュエーションルール)

ノーマライズ後の税引後利益 NI を算出する。

・オーナーの過大役員報酬、親族給与、私的経費等を加算し直す
・本来必要な設備更新コスト等を控除し、保守的に調整する

その上で、以下の価格条件を適用する。

P <= C + k × NI

ここで k は許容する「何年分の利益を前払いするか」を表す係数とする。

推奨レンジ:
・超勝ちレンジ: 1.0 <= k <= 1.5
・勝ちレンジ  : 1.5 <  k <= 2.0

定義:

P_max_super = C + 1.5 × NI
P_max_win   = C + 2.0 × NI

判定:

・P <= P_max_super
→ ラベル: 超勝ち候補
・P_max_super < P <= P_max_win
→ ラベル: 勝ち候補
・P > P_max_win
→ 原則 NG（よほどの成長確信がない限り見送り）

直感的には、「内部留保 + 1〜2年分の利益以内」で買えているかを閾値にする。
	5.	レバレッジ条件 (借入制約)

買収資金として D_new を調達する場合、買収後の D_total と DSCR を用いて安全性を制約する。

利払費用の近似:
Interest ≒ r × D_total

DSCR の近似:
DSCR_approx ≒ E / (r × D_total)

安全側ルール:

・D_new の上限を以下で制約する

D_total <= min( P × 0.7,  E / (r × DSCR_target_hard) )

推奨値:
・DSCR_target_hard = 3.0
・D_new <= P × 0.7 を一つの上限目安とし、それでも DSCR_approx >= 3.0 を満たす範囲に抑える。
	6.	ディール分類ルール

ハード条件 H1〜H6 を満たした前提で、価格条件とレバレッジ条件からラベルを付与する。

Step 1: ハード条件チェック
・H1〜H6 を全て満たしているか。
NGがあれば「見送り」。

Step 2: 価格条件チェック
・P <= P_max_super
→ ラベル候補 A: 超勝ち
・P_max_super < P <= P_max_win
→ ラベル候補 B: 勝ち
・P > P_max_win
→ ラベル: 価格NG → 原則見送り

Step 3: レバレッジ安全性チェック
・希望する D_new を仮置きし、D_total, DSCR_approx を計算
・DSCR_approx < 3.0 になる場合は D_new を削減
・D_new を削ってもディールが成立しない場合は「資金構成上NG」として見送り

最終ラベル:

・ハード条件OK かつ P <= P_max_super かつ DSCR_approx >= 3.0
→ 超勝ち案件候補

・ハード条件OK かつ P_max_super < P <= P_max_win かつ DSCR_approx >= 3.0
→ 勝ち案件候補

・それ以外
→ 見送り対象
	7.	DD時の必須確認ポイント

上記の数式条件を満たしていても、以下に大きな問題があれば NI を下方修正し、再評価する。

D1. ノーマライズの妥当性
・役員報酬、親族給与、私費混入の有無
・本来必要なCAPEX・修繕コストが十分に計上されているか

D2. 設備・更新負担
・設備の老朽化状況、更新投資の近未来負担
・減価償却を削って「見かけ上の利益」を水増ししていないか

D3. 訴訟・保証・オフバランス
・潜在的な訴訟、長期保証義務、瑕疵担保、環境関連リスクなど
・重要な債務や保証がバランスシート外にないか

D4. 人材構造
・現場のキーマン年齢分布（引退リスク）
・オーナー以外のナンバー2、実務責任者の存在とリテンション設計の余地

D5. 許認可の更新条件
・更新サイクルと要件（有資格者常勤数など）
・買収後に要件割れしないか

重大なマイナスが判明した場合:
	1.	NI を保守的に再計算
	2.	P_max_super, P_max_win を再計算
	3.	価格条件を再チェックし、ラベルを更新
	4.	運用フロー (v0.0.1)
	5.	案件情報を収集
・10年分のPL/BS、顧客上位リスト、許認可一覧、年齢構成など
	6.	H1〜H6 をチェック
・一つでもNGなら原則スキップ
・例外検討する場合は別レイヤーで管理
	7.	NI, C をノーマライズして算定
	8.	P_max_super, P_max_win を算出し、Pとの比較でラベル候補を決定
	9.	借入案 D_new を仮置きし、D_total, DSCR_approx を算出
・DSCR_approx >= 3.0 となる範囲で D_new を調整
・調整後も成立しない場合は見送り
	10.	詳細DDで D1〜D5 を検証し、NI・ラベルの修正が必要か判断
	11.	最終的に
・「超勝ち」または「勝ち」ラベル
・DSCR安全
・DDで致命的リスクなし
の三条件を満たす案件のみ、クロージング検討フェーズへ進める。

以上を「推奨M&A v0.0.1」の基準とし、今後は実案件のフィードバックを元に
・kのレンジ
・DSCR_target_hard
・H6の閾値（40%ライン）
などを調整して v0.0.2 以降で更新する。
