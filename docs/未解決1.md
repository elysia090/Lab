REVISED PROBLEM STATEMENT
	1.	UNRESOLVED PROBLEM (THE REAL OPEN ONE)

Input:
	•	a finite alphabet Sigma,
	•	a metric output space (Y, D),
	•	a target pair semantics
Phi : X_infty -> Y,
where
X_infty = { (x,i,j) : x in Sigma*, 1 <= i < j <= |x| }.

Question:
Determine necessary and sufficient conditions for the existence of a family of finite-width exact-monoid summary architectures

A_r = (S_r, e_r, otimes_r, lambda_r, q_r, R_r),   r = 1,2,3,…

such that, for every r,

(A0) width bound:
|S_r| <= r;

(A1) exact monoid summary:
q_r(xy) = q_r(x) otimes_r q_r(y) for all x,y in Sigma*;

(A2) exact factorization through the five summaries:
if x = u a m b v, then
h_r(x,i,j) = R_r(q_r(u), q_r(a), q_r(m), q_r(b), q_r(v));

(A3) logarithmic online operations:
build in O_r(n log n),
append / point update in O_r(log n),
pair query in O_r(log n);

(A4) uniform approximation on each bounded-length slice:
for every fixed N >= 2,
eps_r(N; Phi)
:= sup_{(x,i,j) in X_N} D(h_r(x,i,j), Phi(x,i,j)) -> 0 as r -> infinity,
where
X_N = { (x,i,j) : x in Sigma^{<= N}, 1 <= i < j <= |x| }.

This is the correctly formulated open problem.

It asks for:
	•	exact algebraic compositionality (A1),
	•	exact query factorization (A2),
	•	an explicit finite-width constraint (A0),
	•	logarithmic online operations (A3),
	•	and only approximate agreement with Phi, uniformly on each finite-length slice (A4).

The width bound (A0) is essential.
Without (A0), the problem is trivial: for each r, one may take the bounded exact construction on X_r, which is exact on all lengths <= r, so for every fixed N one has eps_r(N; Phi) = 0 for all r >= N.

Therefore the nontrivial open problem is the finite-width approximation problem, not the unbounded-family problem with no size control.

⸻

	2.	WHAT IS NOW FULLY SOLVED

The bounded exact problem is fully solved.

BOUNDED EXACT PROBLEM

Fix N >= 2, and fix
Phi : X_N -> Y,
where
X_N = { (x,i,j) : x in Sigma^{<= N}, 1 <= i < j <= |x| }.

Question:
Can we construct a finite exact summary architecture on X_N?
Can we compute the unique minimal one?
Can we execute it efficiently online?

Answer:
Yes, completely.

⸻

2.1 FINITE REDUCTION

Define the finite universe

U_N := Sigma^{<= N} union {bot},

where bot is an overflow sink.

Define total bounded concatenation star on U_N by

x star y = xy    if x,y != bot and |xy| <= N,
x star y = bot   otherwise.

Then:
	•	U_N is finite.
	•	star is associative.
	•	empty is the identity.
	•	bot is absorbing.

So the bounded exact problem is reduced to a finite monoid-like algebra
(U_N, star, empty).

⸻

2.2 TOTAL OBSERVATIONAL PROFILES (INCLUDING LEGALITY)

Let T_N be the finite set of all typed 1-hole templates of the five-slot form
(u,a,m,b,v) in which exactly one slot is a hole, the other four slots are fixed,
the a-slot and b-slot are single symbols when fixed, and the total length
constraint <= N is checked after substitution.

For each template C in T_N and each candidate filler w in U_N, define the total observation

Obs(C,w) in Y union {ILL}

as follows:
	•	Obs(C,w) = Phi(C[w]) if substituting w into C yields a legal element of X_N;
	•	Obs(C,w) = ILL otherwise.

Thus legality is encoded explicitly as part of the observation profile.

For each w in U_N, define its total observation profile

Prof(w) := ( Obs(C,w) )_{C in T_N }.

Define the initial equivalence relation ~0 by

w ~0 w’
iff
Prof(w) = Prof(w’).

Put bot in its own singleton class if desired; equivalently, this is already forced by legality behavior.

Interpretation:
w ~0 w’ means that w and w’ are indistinguishable by all typed 1-hole pair observations on X_N, including whether insertion is legal at all.

This is necessary for exactness:
if two strings were placed in the same exact summary state but differed on some one-hole observation, either in value or legality, the readout would fail.

⸻

2.3 WHY OBSERVATION ALONE IS NOT ENOUGH

An exact summary also needs a well-defined merge.
So the state partition must be stable under bounded concatenation.

It is enough to require stability under left/right concatenation by single letters.

For each a in Sigma, define transitions on U_N:

L_a(x) := a star x,
R_a(x) := x star a.

Lemma:
For an equivalence relation E on U_N, the following are equivalent.

(i) E is a two-sided congruence for star:
if x E x’ and y E y’, then x star y E x’ star y’.

(ii) E is stable under all single-letter left/right transitions:
if x E x’, then
L_a(x) E L_a(x’) for all a in Sigma,
and
R_a(x) E R_a(x’) for all a in Sigma.

Proof:

(i) => (ii):
Take one factor to be empty and the other to be the single letter a.
Then congruence immediately gives stability under L_a and R_a.

(ii) => (i):
First show right stability for arbitrary suffixes.
Let y = a_1 … a_k.
Repeated application of single-letter right stability gives

x E x’
=> R_{a_1}(x) E R_{a_1}(x’)
=> …
=> R_{a_k} … R_{a_1}(x) E R_{a_k} … R_{a_1}(x’).

Hence
x star y E x’ star y.

Next, apply the same argument on the left to y E y’:
for fixed x’,
y E y’ implies
x’ star y E x’ star y’.

Now combine the two facts:
	•	from x E x’, we get x star y E x’ star y;
	•	from y E y’, we get x’ star y E x’ star y’.

By transitivity,
x star y E x’ star y’.

Hence E is a two-sided congruence.

So exact merge reduces to stability under finitely many transitions L_a and R_a.

⸻

2.4 THE CANONICAL MINIMAL PARTITION

Define P* to be the coarsest partition of U_N such that:

(P1) P* refines ~0;
(P2) P* is stable under every L_a and R_a.

Because U_N is finite, such a coarsest stable refinement exists and is unique.

This is the standard greatest stable refinement below the observational partition.

⸻

2.5 CONSTRUCTION OF THE EXACT SUMMARY ARCHITECTURE

Let

S := U_N / P*.

For x in U_N, write [x] for its block in S.

Define:

q(x) := [x],
e := [empty],
lambda(a) := [a] for each atomic symbol a in Sigma,

and define merge by

otimes([x],[y]) := [x star y].

Why is otimes well-defined?
Because P* is stable under all L_a and R_a, hence by the lemma it is a two-sided congruence for star.
Therefore x P* x’ and y P* y’ imply x star y P* x’ star y’.

Associativity:
([x] otimes [y]) otimes [z]
= [x star y] otimes [z]
= [(x star y) star z]
= [x star (y star z)]
= [x] otimes [y star z]
= [x] otimes ([y] otimes [z]).

Identity:
[e] otimes [x] = [empty star x] = [x],
[x] otimes [e] = [x star empty] = [x].

So (S, otimes, e) is a finite monoid.

⸻

2.6 DEFINITION OF THE READOUT

For a legal pair decomposition x = u a m b v in X_N, define

R([u],[a],[m],[b],[v]) := Phi(u,a,m,b,v).

We must prove this is well-defined.

Suppose
[u] = [u’], [a] = [a’], [m] = [m’], [b] = [b’], [v] = [v’].

Because P* refines ~0, equality of blocks implies equality of total one-hole observation profiles, including legality.

Now replace the five slots one at a time.

At each step:
	•	the current substitution is legal if and only if the next substitution is legal, because legality is part of the total profile;
	•	when legal, the Phi-value is preserved, again because equality of total profiles includes the observed Phi-value.

Hence

Phi(u,a,m,b,v)
= Phi(u’,a,m,b,v)
= Phi(u’,a’,m,b,v)
= Phi(u’,a’,m’,b,v)
= Phi(u’,a’,m’,b’,v)
= Phi(u’,a’,m’,b’,v’).

Therefore R does not depend on the chosen representatives.
So R is well-defined.

Then exactness is immediate:

Phi(u,a,m,b,v) = R(q(u), q(a), q(m), q(b), q(v)).

Thus an exact summary architecture exists on X_N.

⸻

2.7 MINIMALITY

Take any other exact finite summary architecture on X_N, and let K be the kernel partition of its summary map.

Then:

(i) K refines ~0.
Reason:
If x and x’ were in the same summary state but differed on some typed 1-hole observation, either in legality or in Phi-value, exactness of the readout would fail.

(ii) K is stable under all L_a and R_a.
Reason:
If x and x’ were in the same summary state but a star x and a star x’ landed in different states, merge by the atomic summary lambda(a) would not be well-defined. The same argument applies on the right.

Hence K satisfies (P1) and (P2).
Since P* is the coarsest partition satisfying (P1) and (P2),
K must refine P*.

Therefore every exact architecture has at least as many states as S = U_N / P*.

So the construction from P* is state-minimal.
Because P* itself is unique, the minimal exact quotient is unique up to renaming of states.

⸻

2.8 CONSTRUCTION ALGORITHM

Input model:
	•	Phi is given extensionally on the finite domain X_N,
or equivalently all values Obs(C,w) for typed 1-hole templates can be queried exactly.

Algorithm:

Step 1.
Enumerate U_N = Sigma^{<= N} union {bot}.

Step 2.
Enumerate all typed 1-hole templates T_N.

Step 3.
Compute Prof(w) for every w in U_N and form the initial partition ~0.

Step 4.
Repeatedly refine the current partition whenever some block violates stability under
some L_a or some R_a.

Equivalently:
split any block B whenever there exist x,x’ in B and a in Sigma such that
L_a(x), L_a(x’) land in different current blocks,
or
R_a(x), R_a(x’) land in different current blocks.

Step 5.
When no further split is possible, output the final partition P*.

Step 6.
Build the operation table for otimes via
[x] otimes [y] = [x star y],
and build the readout table R from the exact values of Phi on representatives.

Termination:
The partition lattice on the finite set U_N is finite, and each proper refinement strictly increases the number of blocks.
So the procedure halts.

Correctness:
The fixed point is exactly the coarsest partition satisfying (P1) and (P2), hence P*.

⸻

2.9 COMPUTATIONAL COMPLEXITY

Let

M := |U_N| = 1 + sum_{k=0}^N |Sigma|^k.

Then all objects are finite.

Construction complexity:
	•	The algorithm runs in time polynomial in the extensional input size,
that is, polynomial in the size of the finite tables over U_N and T_N.
	•	This is the correct complexity notion for the bounded extensional problem.

Important:
M is exponential in N in general, so this is not polynomial in N.
But that is unavoidable because the bounded problem itself is presented over an exponentially large finite domain.

Execution complexity after construction:
Once the minimal exact quotient is built,
each node in a balanced tree stores one state in S.
Each internal recomputation is one table lookup for otimes.

Therefore:
	•	build is O(n log n),
	•	append / point update is O(log n),
	•	pair query is O(log n).

So the bounded exact problem is solved both:
	•	semantically (existence and minimality),
	•	algorithmically (construction from finite data),
	•	operationally (logarithmic online use).

	3.	REMAINING OPEN PROBLEMS

What remains open is not the bounded extensional exact problem.
That one is fully solved.

The genuinely remaining open problems are the following.

⸻

3.1 OPEN PROBLEM 1: UNBOUNDED EXACT, EFFECTIVE FORM

Input:
Phi : X_infty -> Y on all lengths.

Abstract semantic existence is already characterized:
a finite exact summary architecture exists if and only if the predictive syntactic congruence has finite index.

What is still open:
for natural intensional representations of Phi
(for example: program, circuit, oracle, probabilistic model),
determine when finite index is decidable,
and when the minimal exact quotient can be effectively constructed.

So the open part is no longer semantic existence;
it is the effective decision/construction problem for natural input classes.

⸻

3.2 OPEN PROBLEM 2: APPROXIMATE FINITE-WIDTH FAMILIES

Fix Phi : X_infty -> Y.

Determine necessary and sufficient conditions for the existence of a family

A_r = (S_r, e_r, otimes_r, lambda_r, q_r, R_r),   r = 1,2,3,…

such that:
	•	|S_r| <= r for every r,
	•	q_r is an exact monoid homomorphism for every r,
	•	online operations remain logarithmic,
	•	and for every fixed N,
eps_r(N; Phi) -> 0 as r -> infinity.

Equivalent restatement:
Which Phi admit exact compositional summaries of width at most r whose induced predictors become uniformly dense on each bounded-length slice?

This is the real revised approximation problem.

This is not solved in general.

⸻

3.3 OPEN PROBLEM 3: RATE OF APPROXIMATION

Define the optimal bounded-length approximation function

E_Phi(N,r)
:= inf_{A_r admissible} sup_{(x,i,j) in X_N} D(h_r(x,i,j), Phi(x,i,j)),

where the infimum ranges over admissible architectures with |S_r| <= r.

Question:
Characterize or bound E_Phi(N,r) in terms of intrinsic structural invariants of Phi.

Candidate invariants include:
	•	approximate predictive congruence covering numbers,
	•	predictive Hankel operator approximation rank,
	•	finite-memory radius,
	•	k-locality,
	•	regularity on a compact quotient,
	•	low-rank predictive state dimension.

This rate problem is genuinely open in general.

⸻

3.4 OPEN PROBLEM 4: ALGORITHMIC SYNTHESIS FOR APPROXIMATION

Given only query access, sample access, or a compact programmatic description of Phi,
and given epsilon > 0,
construct an admissible width-r architecture with error <= epsilon on X_N,
or characterize when this is possible uniformly over bounded-length slices.

Question:
What are the sample complexity, query complexity, time complexity, and required width r = r(epsilon, N)?
More strongly, when can one choose a single width schedule r = r(epsilon) that works uniformly on every fixed bounded-length slice?

This is the algorithmic version of the approximate finite-width problem, and it remains open in general.

⸻

	4.	SHORTEST SUMMARY

Solved exactly:
	•	The bounded exact problem on X_N.
	•	There is always a unique minimal exact quotient.
	•	It is computed by finite partition refinement.
	•	It yields O(log n) online operations after compilation.

Still open:
	•	effective unbounded exact synthesis for natural intensional input classes,
	•	general approximate finite-width existence under an explicit width bound,
	•	approximation rates E_Phi(N,r),
	•	algorithmic synthesis for approximation.

So the precise status is:

bounded exact:
fully solved.

unbounded exact:
semantically characterized, but not fully solved algorithmically in general.

approximate finite-width:
still the main open problem.

One additional correction is essential:

Without the explicit width bound |S_r| <= r, the approximation problem is trivial, because for each r one can use the exact bounded construction on X_r, which gives zero error on all lengths <= r.
Hence the real open problem is the finite-width approximation problem, not the unrestricted family problem.
