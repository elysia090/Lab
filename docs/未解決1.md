
REVISED PROPOSITION:
Well-defined approximate no-regret pair-interaction architecture

This is the correct proposition when exact realizability of Phi is NOT required.
The target may be approximated. Regret is measured only against a fixed
finite-width comparator class induced by the architecture.

----------------------------------------------------------------------
1. Fixed objects
----------------------------------------------------------------------

Fix:
- a finite alphabet Sigma,
- a prediction space K,
- a loss family L, where each round loss ell_t : K -> R is convex,
- a width parameter r.

For any sequence x = x_1 ... x_n and any pair 1 <= i < j <= n, write

    x = u a m b v

with
- u = x_1 ... x_{i-1},
- a = x_i,
- m = x_{i+1} ... x_{j-1},
- b = x_j,
- v = x_{j+1} ... x_n.

----------------------------------------------------------------------
2. Architecture
----------------------------------------------------------------------

A width-r pair-interaction architecture consists of:

- a summary space S,
- an identity element e in S,
- a merge operation

    otimes : S x S -> S,

- an atomic summary map

    lambda : atomic blocks -> S,

- a convex parameter space Theta,

- a readout family

    R_theta : S x S x S x S x S -> K
    for theta in Theta,

- an online update rule

    U : Theta x G -> Theta,

  where G is the subgradient space.

For a decomposition x = u a m b v, the prediction is

    yhat_theta(x; i, j)
      = R_theta( s(u), s(a), s(m), s(b), s(v) ),

where s(.) is the interval summary induced by lambda and otimes.

The induced comparator class is

    H_r
      = { h_theta : (x, i, j) |-> yhat_theta(x; i, j)  :  theta in Theta }.

----------------------------------------------------------------------
3. What "well-defined approximate no-regret" means
----------------------------------------------------------------------

The architecture is called a well-defined approximate no-regret architecture if:

(A) Representation invariance:
    For every sequence, every valid balanced-tree representation, every valid
    O(log n) interval cover, and every legal edit sequence, the prediction,
    loss, and next parameter depend only on the semantic input (x, i, j, theta),
    not on tree shape, parenthesization, or cover choice.

(B) Comparator invariance:
    The hypothesis class H_r is fixed by the architecture itself, and does not
    depend on accidental internal representations.

(C) Valid online convex optimization layer:
    The per-round optimization problem over Theta is well-posed, so regret with
    respect to H_r is meaningful.

(D) Sublinear regret target:
    For every valid round sequence, the learner can be analyzed against H_r via

        Regret_T
          = sum_{t=1}^T ell_t(yhat_t)
            - inf_{theta in Theta} sum_{t=1}^T ell_t( yhat_theta(x^(t); i_t, j_t) )

    and the intended guarantee is Regret_T = o(T)
    (ideally O(sqrt(T)) under the usual boundedness assumptions).

Note:
No exact semantic object Phi is required to lie inside H_r.

----------------------------------------------------------------------
4. Proposition (necessary and sufficient conditions)
----------------------------------------------------------------------

A width-r pair-interaction architecture is a well-defined approximate no-regret
architecture if and only if all of the following hold.

(W1) Summary monoid
-------------------
There exists e in S such that (S, otimes, e) is a monoid:

- associativity:
    (x otimes y) otimes z = x otimes (y otimes z)

- identity:
    e otimes x = x = x otimes e

and the interval summary map s is the unique homomorphic extension of lambda:

    s(IJ) = s(I) otimes s(J)

for every concatenation of adjacent intervals I, J.

This is necessary and sufficient for summary values to be independent of tree
shape and parenthesization.

(W2) Query factorization through summaries
------------------------------------------
For every theta in Theta, the prediction for any pair query factors only through
the five summaries:

    yhat_theta(x; i, j)
      = R_theta( s(u), s(a), s(m), s(b), s(v) ).

Equivalently, if two valid decompositions yield the same five summaries, then
they must yield the same prediction.

This is necessary and sufficient for query answers to be independent of cover
choice and internal representation.

(W3) Fixed nonempty comparator class
------------------------------------
The set

    H_r = { h_theta : theta in Theta }

is nonempty and depends only on (S, otimes, lambda, Theta, R), not on any
particular tree encoding.

This is necessary and sufficient for regret to be measured against a fixed
semantic comparator class, rather than against moving implementation artifacts.

(W4) Representation-invariant local edit update
-----------------------------------------------
Every legal append or point update changes only O(1) atomic blocks, and the new
global summaries are obtained only by:

- replacing the affected atomic summaries, and
- recomputing ancestors via the same otimes.

Equivalently, two internal representations of the same old sequence, after the
same semantic edit, must induce the same new root summary.

This is necessary and sufficient for the data-structure state transition to be
well-defined.

(W5) Convex parameter layer
---------------------------
Theta is a nonempty convex set, and for every fixed summary tuple

    z = ( s(u), s(a), s(m), s(b), s(v) ),

the map

    theta |-> ell_t( R_theta(z) )

is convex on Theta for every allowed round loss ell_t.

This is necessary and sufficient for the regret objective against H_r to be a
valid online convex optimization problem.

(W6) Update depends only on summary-level subgradient information
----------------------------------------------------------------
The parameter update rule U depends only on:

- the current parameter theta_t, and
- a valid subgradient of theta |-> ell_t( R_theta(z_t) ),

and not on any lower-level accidental representation.

This is necessary and sufficient for the learning dynamics to be
representation-invariant and to define a single optimizer on Theta.

----------------------------------------------------------------------
5. Consequence
----------------------------------------------------------------------

If (W1)-(W6) hold, then:

- predictions are representation-invariant,
- the comparator class H_r is fixed and meaningful,
- edits induce a unique next summary state,
- learning is a well-posed online convex optimization problem over Theta.

If the balanced tree has height O(log n), then the architecture supports:

- build in O(n log n),
- append / point update in O(log n),
- pair query in O(log n),

and standard online convex optimization algorithms may be applied on Theta.

Any comparison to an external target Phi then decomposes into:

    total excess loss
      = approximation error of H_r relative to Phi
        + regret inside H_r.

Thus exact realizability of Phi is NOT required for the architecture to be
well-defined as a no-regret learner.

----------------------------------------------------------------------
6. Why this is an iff statement
----------------------------------------------------------------------

(=> necessity)

If the architecture is well-defined for approximate no-regret learning, then:

- representation-invariant summaries force associativity and identity, so (W1);
- representation-invariant query answers force summary-only readout, so (W2);
- meaningful regret requires a fixed nonempty comparator class, so (W3);
- representation-invariant edits force local summary recomputation, so (W4);
- meaningful convex regret minimization requires convexity in theta, so (W5);
- representation-invariant learning dynamics force subgradient-only parameter
  updates at the summary level, so (W6).

(<= sufficiency)

If (W1)-(W6) hold, then:

- (W1) makes every interval summary unique up to semantics;
- (W2) makes every prediction depend only on semantic summaries;
- (W3) fixes the comparator class H_r independent of implementation details;
- (W4) makes edits induce a unique next summary state;
- (W5) makes each round a valid convex optimization problem on Theta;
- (W6) makes the optimizer itself representation-invariant.

Therefore the full prediction-update process is semantically unique, and regret
against H_r is well-defined.

----------------------------------------------------------------------
7. What remains open
----------------------------------------------------------------------

The real open problem is not the iff above.

The real open problem is:

For which pair-interaction semantics Phi does there exist a finite-width summary
space S and readout family R such that:
- (W1)-(W6) hold,
- build / update / query are logarithmic,
- and the approximation error of H_r relative to Phi is small or vanishes as r grows?

That is the actual approximation-theoretic existence theorem still missing.
